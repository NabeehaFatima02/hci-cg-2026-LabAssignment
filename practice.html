    ✅ True

    ✅ True

Part C: Short Answer — Answers

Q1) Difference between a semantic element and a generic element

Semantic element (e.g., <nav>, <header>, <article>):
These elements clearly describe their meaning and purpose. They improve accessibility, SEO, and help screen readers understand the page structure.
Example: <nav> tells that the content is for navigation.

Generic element (e.g., <div>, <span>):
These have no inherent meaning. They are mainly used for styling or layout purposes when no semantic element fits.
Example: <div> just groups content without explaining what it is.

Q2) Why does KLM not model high-level planning and interpretation?

KLM (Keystroke-Level Model) focuses only on low-level physical actions like keystrokes, mouse movements, and mental preparation time.

It assumes the user is already skilled and knows what to do, so it does not include thinking, decision-making, or understanding tasks.

High-level planning and interpretation vary from user to user, so KLM ignores them to keep predictions simple and consistent.
