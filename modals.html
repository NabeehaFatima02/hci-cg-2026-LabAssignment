Norman’s Action Cycle in HCI (Chapter 3)

Definition:
In Human–Computer Interaction, norms refer to the accepted rules, standards, or expectations that guide how users interact with computer systems. Norman’s Action Cycle helps users understand what behaviour is expected when using an interface.

Stages of Norman’s Action Cycle:
Norman’s Action Cycle consists of the following stages:

  1. Goal Formation:
    The user decides what they want to achieve (e.g., saving a file).
  2. Intention Formation:
    The user plans how to achieve the goal (e.g., clicking “Save”).
  3. Action Execution:
    The user performs the action using the interface.
  4. System Response:
    The system reacts to the user’s action.
  5. Perception:
    The user observes the system’s response.
  6. Interpretation:
    The user understands what the response means.
  7. Evaluation:
    The user checks whether the goal has been achieved.

Design Note:
Good interface design follows norms so that users do not feel confused or frustrated.

GOMS Model in HCI (Chapter 12, Page# 422-424)

What is the GOMS Model?
GOMS is a predictive model used in Human–Computer Interaction to analyze how users perform tasks when interacting with a computer system. It helps designers estimate task completion time and understand the steps users follow to achieve a goal.

GOMS stands for:

  • Goals
  • Operators
  • Methods
  • Selection Rules

Components of the GOMS Model:

  • Goals:
    Goals represent what the user wants to achieve.
    Example: Sending an email, saving a document.
  • Operators:
    Operators are the basic actions performed by the user, such as keystrokes, mouse movements, clicking, and mental actions.
  • Methods:
    Methods are the sequences of steps used to achieve a goal. A goal may have multiple methods.
    Example: Saving a file using a keyboard shortcut or a menu option.
  • Selection Rules:
    Selection rules determine which method is chosen when more than one method is available.
    Example: Use Ctrl + S if the shortcut is known; otherwise use the menu.

Importance of GOMS in HCI:

  • Predicts user performance time
  • Helps compare interface designs
  • Useful for expert users
  • Identifies inefficient interaction steps

Limitations of GOMS:

  • Assumes expert, error-free users
  • Does not model learning or mistakes
  • Not suitable for exploratory or creative tasks
